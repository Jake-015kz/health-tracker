## 2. Убрать дублирование: Sidebar vs Top Tabs

**Проблема:** На десктопе одновременно видны Sidebar (слева) И Tab-бар внутри Dashboard (вверху контента). Оба управляют одной и той же навигацией. Это дублирование.

**Решение:** Убрать Tab-бар из Dashboard на десктопе, оставить только Sidebar как навигацию. На мобилке — BottomNav + select (как сейчас).

### Файлы:
- **`src/app/dashboard/page.tsx`** — убрать рендер `.tabsDesktop` (строки 78-88), оставить только `select` для мобилки
- **`src/app/dashboard/page.module.css`** — удалить `.tabsDesktop` стили (строки 5-20)
- **`src/widgets/layout/sidebar/sidebar.tsx`** — починить навигацию: сейчас sidebar ссылки не синхронизируются с табами dashboard, т.к. sidebar ссылки ведут на `/dashboard#log` а dashboard page читает hash через `getTabFromHash()`. Проблема в том, что если мы уже на /dashboard и кликаем sidebar ссылку с другим hash — Next.js не перезагружает страницу, hashchange event не всегда срабатывает корректно с Next.js Link. Нужно переписать sidebar навигацию на `useRouter().push()` с принудительным scroll/refresh.

### Исправление sidebar навигации:
```tsx
// Вместо <Link> использовать обработчик клика:
const router = useRouter();
const handleClick = (e: React.MouseEvent, item: typeof NAV_ITEMS[number]) => {
  e.preventDefault();
  if (item.href === pathname) {
    // Уже на dashboard — просто меняем hash
    window.history.replaceState(null, "", item.href);
    window.dispatchEvent(new HashChangeEvent("hashchange"));
  } else {
    router.push(item.href);
  }
};
```

## 7. Расписание приёма лекарств (Daily Schedule + Ad-hoc)

### Концепция

Текущая система `Medication` + `MedicationLog` УЖЕ работает как шаблон:
- Каждое лекарство имеет `frequency: MedicationTime[]` — это и есть "настройка на сутки"
- `MedicationLog` отслеживает что принято каждый день
- Шаблон автоматически повторяется каждый день

**Чего не хватает:**
1. **Ad-hoc лекарства** — добавить таблетку на сегодня (от высокого давления) без сохранения в шаблон
2. **Группировка по времени** — показать утро/день/вечер отдельными секциями
3. **Логика "ИЛИ"** — Глюконил ИЛИ Глюкованс (сейчас это два отдельных лекарства)

## 8. Баг-фиксы и улучшения

**Решение:** Вызвать `migrateLocalStorage()` после загрузки данных из Supabase, если есть данные в localStorage.

**Файл:** `src/features/auth/model/use-data-store.ts`

## 9. Критический баг: данные не попадают в Supabase

### Проблема
Все `insert` в `useDataStore` (biometric_entries, medications, medication_logs) **не проверяют ошибки**. Если insert фейлится (RLS policy, foreign key violation) — данные всё равно добавляются в React state и отображаются, но в Supabase ничего не сохраняется. На другом устройстве данные пустые.

### Корень
`addBiometric`, `addMedication`, `toggleMedLog` — все делают `await supabase.from(...).insert(...)` без проверки `.error`.

### Решение
1. Проверять `error` после каждого insert/update/delete
2. При ошибке — показать toast/alert пользователю и НЕ обновлять React state
3. Добавить retry или хотя бы console.error

### Файлы
- `src/features/auth/model/use-data-store.ts` — все CRUD функции

### Верификация
1. Добавить измерение → проверить в Supabase Dashboard что строка появилась
2. Ошибка RLS → пользователь увидит сообщение
